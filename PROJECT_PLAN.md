# AI Gallery — 项目计划书

> 版本：v0.1 | 创建日期：2026-07-01 | 状态：进行中

---

## 一、项目简介

**AI Gallery** 是一款运行在 Android 14+ 设备上的本地相册应用。  
核心理念：**本地优先、隐私安全、AI 可选**。

- 所有基础功能（浏览、分类、搜索）完全离线可用。  
- AI 功能（智能标签、语义搜索、场景识别）为**用户自备 Key 的可选模块**，未配置时应用正常运行，AI 入口显示引导态而非报错。

## 二、开发阶段规划

### 阶段零：工程基建（当前阶段）
| 子步骤 | 内容 | 状态 |
|--------|------|------|
| 0-1 | 生成架构文档（本文件 + ARCHITECTURE.md） | ✅ 已完成 |
| 0-2 | 配置 build.gradle.kts，锁定核心依赖版本 | ⬜ 待确认 |

### 阶段一：基础相册功能
- MediaStore 读取本地图片/视频
- 相册列表页、媒体网格页、媒体详情页

### 阶段二：增强浏览体验
- 视频播放（Media3/ExoPlayer）
- 分页加载（Paging3）
- 本地收藏/标记（Room）

### 阶段三：AI 可选模块
- 设置页 AI 配置入口（Base URL + API Key）
- 连通性测试
- 智能标签、场景描述（调用用户自配置的 API）
- 语义搜索

### 阶段四：打磨与发布
- 深色模式、动态主题（Material You）
- 性能优化、内存管理
- 打包签名、发布准备

## 三、目标设备与 SDK 要求

| 项目 | 值 |
|------|----|
| 最低 SDK | **34（Android 14）** |
| 目标 SDK | **35（Android 15）** |
| 开发语言 | Kotlin |
| UI 框架 | Jetpack Compose |
| 最低内存 | 3 GB RAM（建议 4 GB+）|

---

## 四、核心约束（红线，不可违反）

1. **禁止硬编码任何 AI API Key 或 Base URL**，一律由用户在设置页自行填写。  
2. **AI 功能永远是可选模块**，未配置时基础功能 100% 离线可用。  
3. **UI 层禁止直接操作数据库或网络**，所有数据访问必须经过 Repository。  
4. **所有代码必须有清晰中文注释**。  
5. **小步迭代**，每个子任务完成后暂停，等待人工确认后再继续。

---

## 五、验收方式

每个阶段由开发者（AI）完成代码后，个人开发者在真机上执行以下操作验收：

- **阶段零**：用 Android Studio 能正常 Build，无编译报错。
- **阶段一**：真机可看到本地相册图片列表，点击可放大查看。
- **阶段二**：视频可播放，长列表滑动流畅无卡顿。
- **阶段三**：填写 API Key 后 AI 标签生效；不填 Key 时入口显示"配置 AI 功能"引导，不报错崩溃。
- **阶段四**：深色模式正常切换，APK 可安装到干净设备。
